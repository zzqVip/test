Number of participants: —

这个数字是什么：它是第一个负载客户端在Jitsi/XMPP 会议里看到的总人数（始终包含你自己这一路）。 所以会议室里「只有你／只有这一路负载」时，正常就是 1，不是 0；要对照「还有几个其他人」，用这个数减去已进入会议的本地负载客户端数量（或由控制台日志判断）。这不是 Gaea 后台单独的参会列表。

只有你在线上链接里已经真正进了会（过了会前预览），且负载端确实进了同一个房间时，这里通常会≥2。

libs/lib-jitsi-meet.min.js 若 404：必须从你们 Jitsi Meet 构建产物拷贝该文件到本仓库 libs/，否则会无法正常初始化 WebRTC/XMPP（不要用磁盘缓存误以为能用）；只有 load-test-participant.min.js 不够。

入会 JWT：每个客户端会先请求 POST …/api/v1/meetings/<会议 id>/join-guest， displayName 为随机 mock（形如 loadtest-p0-…），再用响应里的 jitsiJwt 入会；不需要在 URL 上带 token。 若控制台出现 CORS 错误，请让 API 允许当前页面 Origin（本地为 http://localhost:…）或将负载页部署到 meeting.gaea-labs.com 同源。

示例 URL：http://localhost:9090/meeting/cmoxw20dd0007qm0d3ffepgk8?numClients=5&clientInterval=300 （改代码后务必 npm run build，浏览器强制刷新避免旧 JS 缓存）。打开开发者工具 Console，应看到一行 Load-test: room=… numClients=…；若 numClients 仍是 1，说明参数未进到脚本。

勿对未授权的生产环境开大并发。
